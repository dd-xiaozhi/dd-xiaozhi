# 🎨 GitHub Profile README 使用说明

## 📁 项目文件说明

```
github_home_page/
├── README.md              # 完整版 Profile README (推荐使用)
├── README_SIMPLE.md       # 简洁版 Profile README (备选)

│   └── workflows/
│       └── snake.yml      # 贡献蛇动画的 GitHub Actions 配置
└── 使用说明.md            # 本文件
```

## 🚀 快速开始 (3步完成)

### 第 1 步: 创建特殊仓库

1. 访问 GitHub: https://github.com/new
2. 仓库名称填写: **你的 GitHub 用户名** (必须完全一致!)
   - 例如: 如果你的用户名是 `xiaozhi`, 就填 `xiaozhi`
3. ✅ 勾选 "Add a README file"
4. 选择 Public (公开)
5. 点击 "Create repository"

> 💡 GitHub 会提示: "You found a secret! This is a special repository..."

### 第 2 步: 上传文件

#### 方式 A: 通过 GitHub 网页 (简单)

1. 打开你刚创建的仓库
2. 点击 "Add file" → "Upload files"
3. 上传 `README.md` 和 `.github/workflows/snake.yml`
4. 提交更改

#### 方式 B: 通过 Git 命令 (推荐)

```bash
# 克隆你的新仓库
git clone https://github.com/YOUR_USERNAME/YOUR_USERNAME.git
cd YOUR_USERNAME

# 复制文件
cp /Users/xiaozhi/work/web/github_home_page/README.md .
cp -r /Users/xiaozhi/work/web/github_home_page/.github .

# 提交并推送
git add .
git commit -m "✨ Add awesome profile README"
git push
```

### 第 3 步: 修改个人信息

打开 `README.md`, 替换以下内容:

```markdown
YOUR_USERNAME      → 你的 GitHub 用户名 (全局替换)
你的名字           → 你的真实姓名或昵称
your.email@example.com → 你的邮箱
PROJECT_NAME_1     → 你的项目名称 1
PROJECT_NAME_2     → 你的项目名称 2
```

**快速替换方法** (使用编辑器的查找替换功能):
- 查找: `YOUR_USERNAME`
- 替换为: `你的实际用户名`
- 全部替换

## 🔧 自定义配置

在 README.md 中查找 `theme=tokyonight`, 替换为:

**暗色主题:**
- `tokyonight` - 东京之夜 (默认)
- `dracula` - 德古拉

**亮色主题:**

### 2. 修改技能标签

在 "🛠️ 技术栈" 部分添加或删除:

```markdown

```

**常用图标名称:**
- python, javascript, typescript, java, go, rust, php

### 3. 添加精选项目

找到 "📌 精选项目" 部分,复制这段代码:

```markdown

```

替换 `REPO_NAME` 为你的项目名称。

### 4. 隐藏某些语言

如果你想在语言统计中隐藏某些语言(比如 HTML/CSS):

```markdown

```

## 🐍 启用贡献蛇动画

### 步骤:

1. 确保 `.github/workflows/snake.yml` 文件已上传

2. 在仓库页面点击 "Actions" 标签

3. 如果看到提示,点击 "I understand my workflows, go ahead and enable them"

4. 点击左侧 "Generate Snake" workflow

5. 点击右侧 "Run workflow" → "Run workflow" 手动运行一次

6. 等待 1-2 分钟,workflow 完成后刷新你的主页

### 故障排除:

如果蛇动画不显示:
- 检查 Actions 是否启用
- 查看 workflow 运行日志
- 确保仓库是 Public
- 等待几分钟后刷新页面

## 🔍 常见问题

### Q1: 为什么统计卡片不显示?

**A:** 可能原因:
1. 用户名拼写错误
2. 仓库是私有的
3. 服务器暂时不可用

**解决:** 检查用户名,等待几分钟后刷新

### Q2: 语言统计不准确怎么办?

**A:** GitHub 的语言检测基于文件扩展名,可能不完全准确。

**解决:** 
- 使用 `hide=language1,language2` 隐藏某些语言
- 在项目中添加 `.gitattributes` 文件调整

### Q3: 如何更新主页内容?

**A:** 直接编辑 README.md 文件并提交即可,GitHub 会自动更新。

### Q4: 可以设置为私有仓库吗?

**A:** 不可以,Profile README 必须是公开仓库才能显示。

### Q5: 统计数据多久更新一次?

**A:** GitHub Stats 卡片会缓存数据,通常每几小时更新一次。
